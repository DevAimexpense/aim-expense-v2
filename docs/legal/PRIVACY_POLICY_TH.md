## 1. ขอบเขต

นโยบายนี้อธิบายวิธีที่ Aim Expense เก็บ ใช้ เปิดเผย และคุ้มครองข้อมูลส่วนบุคคล
ตามพระราชบัญญัติคุ้มครองข้อมูลส่วนบุคคล พ.ศ. 2562 (PDPA) สำหรับผู้ใช้บริการเว็บแอป
และ LINE Official Account ของ Aim Expense

## 2. หลักการสำคัญ — Data Sovereignty

**ข้อมูลธุรกิจของท่าน (รายการจ่าย, ใบเสร็จ, ลูกค้า, ใบเสนอราคา, ใบวางบิล) เก็บไว้ใน
Google Drive และ Google Sheets ของบัญชี Google ที่ท่านเชื่อมต่อ — ไม่ได้เก็บไว้
ในเซิร์ฟเวอร์ของเรา**

ท่านสามารถเข้าถึง ดาวน์โหลด หรือลบข้อมูลตัวเองได้ตลอดเวลาผ่าน Google Drive โดยตรง
(แม้ไม่ผ่านแอป Aim Expense)

ระบบของเราเก็บเพียง **infrastructure metadata** เพื่อให้บริการได้ (รายละเอียดในข้อ 3)

## 3. ข้อมูลส่วนบุคคลที่เราเก็บ

### 3.1 บัญชีผู้ใช้ (User account)
- อีเมลและชื่อจาก Google Sign-In หรือ LINE Login
- LINE User ID, ชื่อโปรไฟล์ LINE, รูปโปรไฟล์ LINE (สำหรับผู้ใช้ที่ login ผ่าน LINE)
- OAuth tokens ของ Google (เข้ารหัส AES-256 ก่อนเก็บ)
- หมายเลขโทรศัพท์ (ถ้ากรอกในโปรไฟล์)

### 3.2 องค์กร (Organization)
- ชื่อบริษัท / ชื่อองค์กร
- ที่อยู่ (ถ้ากรอก)
- เลขประจำตัวผู้เสียภาษี (ถ้ากรอกใน org settings)
- รหัส Google Spreadsheet ID + Drive Folder ID (อ้างถึงไฟล์ในบัญชี Google ของท่าน)

### 3.3 ข้อมูลการสมัครสมาชิก (Subscription)
- แผนการใช้งาน (free / basic / pro / business / max / enterprise)
- ประวัติการชำระเงิน (ผ่าน Stripe — ดูข้อ 6)

### 3.4 บันทึกการใช้งาน (Audit log)
- การกระทำในระบบ (create / update / delete / approve) พร้อม **อ้างอิงเฉพาะ ID และ
  สรุปสั้น** — **ไม่มีจำนวนเงิน, ชื่อร้านค้า, หรือเนื้อหาใบเสร็จ**

### 3.5 บันทึกทางเทคนิค
- IP address, browser user-agent, request timestamps (เก็บใน Vercel logs สูงสุด 30 วัน)
- Cache ของแถว Sheets ใน Upstash Redis (TTL 30 วินาที — อาจมี PII ของลูกค้า/ผู้รับเงิน
  ในรูปข้อความชั่วคราว)

### 3.6 ข้อมูลที่เราไม่เก็บ
- เนื้อหาใบเสร็จ, ใบกำกับภาษี, สำเนาบัตรประชาชน, ที่อยู่ของลูกค้า/ผู้รับเงิน — เก็บใน
  Google Sheets/Drive ของท่านเท่านั้น
- ผล OCR หรือผลวิเคราะห์ AI — ส่งกลับให้ frontend ทันทีโดยไม่บันทึกในฐานข้อมูล
- รหัสผ่าน — ใช้ OAuth (Google / LINE) เท่านั้น ไม่มีระบบรหัสผ่าน

## 4. แหล่งที่มาของข้อมูล

| แหล่ง | ข้อมูล |
|------|--------|
| ท่านกรอกเอง | โปรไฟล์, ชื่อบริษัท, ลูกค้า/ผู้รับเงิน |
| Google OAuth | อีเมล, ชื่อ, OAuth tokens |
| LINE Login | LINE User ID, ชื่อ, รูปโปรไฟล์ |
| LINE Messaging API | ข้อความและรูปใบเสร็จที่ท่านส่งมา (ผ่าน LINE OA) |
| ระบบสร้างเอง | ID, timestamps, audit log |

## 5. วัตถุประสงค์และฐานทางกฎหมาย

| วัตถุประสงค์ | ฐานทางกฎหมาย |
|------------|----------------|
| ให้บริการตามสัญญาการใช้งาน | สัญญา (มาตรา 24(3) PDPA) |
| เก็บ audit log สำหรับความปลอดภัย | ประโยชน์โดยชอบด้วยกฎหมาย (มาตรา 24(5)) |
| สื่อสารทาง LINE Official Account | ความยินยอม (มาตรา 24(1)) |
| ส่งใบกำกับภาษี / รายงานภาษี | ปฏิบัติตามกฎหมาย (มาตรา 24(6)) |
| OCR ใบเสร็จเพื่อช่วยกรอกข้อมูล | สัญญา + ความยินยอม (เปิด/ปิดได้) |
| คิดเงิน subscription | สัญญา |

## 6. ผู้ประมวลผลข้อมูล (Sub-processors)

ดูรายชื่อเต็มและสถานที่จัดเก็บข้อมูลใน [SUB_PROCESSORS.md](SUB_PROCESSORS.md). โดยสรุป:

- **Google (USA)** — Sheets, Drive, OAuth
- **Supabase (Singapore)** — Postgres เก็บ user/org/subscription metadata
- **Vercel (USA/Singapore edge)** — hosting + serverless
- **LINE (ญี่ปุ่น)** — Login + Messaging API
- **Upstash (Singapore)** — Redis cache (TTL 30 วินาที)
- **AksonOCR (Thailand)** — OCR ใบเสร็จไทย (primary)
- **OpenAI (USA)** — GPT-4o OCR fallback (เปิด/ปิดได้)
- **Stripe (USA, อนาคต)** — เก็บเงิน subscription

## 7. การโอนข้อมูลข้ามประเทศ

ข้อมูลถูกประมวลผลใน **Singapore (Vercel + Supabase + Upstash), USA (Google + OpenAI),
ญี่ปุ่น (LINE), ไทย (AksonOCR)**. การโอนนอกประเทศไทยอาศัยมาตรการคุ้มครองตามมาตรา 28 PDPA
(Standard Contractual Clauses ของ sub-processor แต่ละราย).

## 8. ระยะเวลาการเก็บ

| ข้อมูล | ระยะเวลา |
|-------|---------|
| User profile + Org metadata | จนกว่าท่านจะลบบัญชี |
| OAuth tokens | จนกว่าท่านจะ revoke |
| Subscription + invoice | 7 ปี (ตาม พรบ. การบัญชี) |
| Audit log | 1 ปี |
| Vercel access logs | 30 วัน |
| Redis cache | 30 วินาที (TTL ตามนโยบาย) |
| ข้อมูลใน Google Sheets/Drive ของท่าน | ท่านควบคุมเองตลอดเวลา |

## 9. สิทธิของเจ้าของข้อมูล (Data Subject Rights)

ท่านมีสิทธิตามมาตรา 30-37 PDPA:

- **เข้าถึง / ขอสำเนา** ข้อมูลส่วนบุคคล
- **แก้ไข** ข้อมูลที่ไม่ถูกต้อง
- **ลบ** ข้อมูล / **ลบบัญชี**
- **ระงับการประมวลผล** ชั่วคราว
- **คัดค้าน** การประมวลผลโดยเฉพาะการตลาด
- **ขอย้ายข้อมูล** (data portability)
- **เพิกถอนความยินยอม** (ที่ให้ไว้)
- **ร้องเรียน** ต่อ คณะกรรมการคุ้มครองข้อมูลส่วนบุคคล (PDPC) หากเห็นว่าเราไม่ปฏิบัติตาม PDPA

ใช้สิทธิได้ที่: **/account/data** ในแอป หรือส่งอีเมลถึง **dpo@aimexpense.com**
(ตอบกลับภายใน 30 วันตาม PDPA)

## 10. มาตรการความปลอดภัย

- HTTPS/TLS 1.2+ ทุกการเชื่อมต่อ
- OAuth tokens เข้ารหัส AES-256 ก่อนเก็บใน Postgres
- ใช้ Google OAuth / LINE Login (ไม่เก็บรหัสผ่าน)
- สิทธิการเข้าถึงแบบ role-based + permission keys
- Audit log ทุก mutation
- เซิร์ฟเวอร์ Singapore (Supabase + Upstash) — ลดความหน่วง + เป็นภูมิภาคปลอดภัย

## 11. การแจ้งเหตุละเมิด (Breach Notification)

เราจะแจ้ง คณะกรรมการคุ้มครองข้อมูลส่วนบุคคล ภายใน **72 ชั่วโมง** เมื่อพบเหตุละเมิดที่
มีความเสี่ยงต่อสิทธิของเจ้าของข้อมูล. หากเหตุละเมิดมีความเสี่ยงสูง เราจะแจ้งเจ้าของ
ข้อมูลที่ได้รับผลกระทบโดยตรงด้วย ตามมาตรา 37(4) PDPA.

ดูขั้นตอนการตอบสนองใน [INCIDENT_RESPONSE.md](INCIDENT_RESPONSE.md).

## 12. คุกกี้และการติดตาม

- **Session cookies** — สำหรับ login (จำเป็น)
- **Vercel Analytics** — สถิติการใช้งาน (ไม่มี PII, ไม่ track cross-site)
- **Sentry** — error tracking (ไม่มีเนื้อหาใบเสร็จ)
- **ไม่มี** advertising / behavioral tracking cookies
