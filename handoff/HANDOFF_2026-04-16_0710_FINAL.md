## 📜 BUSINESS RULES (ห้ามลืม!)

### Payment Permissions
```
สถานะ pending/rejected → user ที่มี updatePayments แก้ได้
สถานะ approved/paid/cleared → ต้องมี editPaymentAfterApproval (admin/manager เท่านั้น)
```

### Substitute Receipt (ใบรับรองแทนใบเสร็จ)
แสดงปุ่ม 🧾 เฉพาะ:
- vatAmount === 0
- !invoiceFileUrl (ไม่มีใบแจ้งหนี้/ใบเสนอราคา)
- !receiptUrl (ยังไม่มีใบเสร็จ)
- (WTH มี/ไม่มี ไม่สำคัญ)

### Multi-Org
- 1 LINE = N orgs (schema รองรับ)
- Login → ถ้ามีหลาย org → `/select-org` picker (ปลอดภัย vs Paypers dropdown)
- Invited user (non-admin) → ใช้ Google ของ admin owner, ไม่ต้อง setup
- Owner (สร้าง org เอง) → onboarding google + company

### Approval Flow
```
สร้างรายจ่าย (status=pending)
  ↓
/approvals → batch select → "✅ อนุมัติ + เลือกวันจ่าย" → status=approved + PaymentDate set
  ↓
/payment-prep → group by payee+paymentDate → mark paid → status=paid
  ↓
/payments → upload receipt (Stage 2) → status=paid + receiptUrl
```

### WTH (หัก ณ ที่จ่าย)
- Master list: `WTH_TYPES` ใน `lib/wth-types.ts`
- ทั้ง UploadInvoiceModal + PaymentModal ใช้ dropdown เดียวกัน
- มี "กำหนดเอง" สำหรับ rate ที่ไม่อยู่ใน list
