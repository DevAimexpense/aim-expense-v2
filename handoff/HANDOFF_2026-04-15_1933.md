## 📝 Session Log Entry — 2026-04-15 17:23 — 🚨 CRITICAL BUG FIX

**Discovered:** User report — ค่าที่บันทึกใน Sheet **ผิดทุก field**
- User input: ค่าอาหาร, 2000, WTH 3%, VAT
- Sheet shows: Description="CB-MNZP1MM7-TOTW", WTH 1%=฿1, VAT ฿2000, GTTL ฿3, DueDate "01 ม.ค. 23"

**Root cause:**
- `appendRow()` เขียนเป็น array เรียงตาม index ตาม SHEET_HEADERS ใน code
- `getAll()` อ่าน headers จาก Sheet จริง (rows[0]) แล้ว map ด้วย header name
- ถ้า Sheet column order ต่างจาก code → write/read ไม่ตรงกัน → ค่าไปลงผิดช่อง
- **User's Sheet ถูกสร้างมาก่อน มี column order ต่างจาก code** → bug

**Fix:**
- เพิ่ม `appendRowByHeaders(tabName, obj)` ใน `google-sheets.service.ts`
  - อ่าน headers จาก Sheet จริงก่อน → map input object เข้ากับ column order จริง
  - ถ้า Sheet ขาด column ใด → skip (ไม่ทำ error)
  - Boolean → "TRUE"/"FALSE" auto-convert
- Refactor `payment.router.ts create` → ใช้ `appendRowByHeaders` แทน `appendRow(array)`
- ✅ TypeScript typecheck ผ่าน

**Note เรื่อง update:**
- `updateById` ใช้ header-name mapping อยู่แล้ว → ไม่พัง
- ยืนยันจาก user: "แก้ไขรายการได้แล้ว" (ค่าถูก) → พิสูจน์ update OK แล้ว
- มีแค่ create ที่พัง — ซึ่ง fix แล้วในรอบนี้

**Action required from user:**
1. `rm -rf .next && npm run dev` restart
2. ลบ/แก้รายการเก่า 2 รายการที่ข้อมูลเพี้ยน (แก้ → save → ค่าถูกเพราะ update ใช้ header name)
3. ทดสอบสร้างรายการใหม่: ค่าอาหาร 2000 WTH 3% VAT → ค่าใน Sheet ต้องถูก

**Lesson learned:**
- เวลา write Google Sheet → **ใช้ header-name mapping เสมอ** ไม่ใช่ positional array
- ตรวจสอบ write/read ใช้วิธีเดียวกัน ถ้าไม่ → misalignment bug ซ่อนยาก
- เหลือ `appendRow(array)` ที่อื่น ๆ ในโค้ด — ต้องไล่ refactor ทั้งหมดในอนาคต
