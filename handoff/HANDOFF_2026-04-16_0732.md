### Flow ทำงาน

```
User ส่งรูปใบเสร็จใน LINE OA
  ↓
webhook: handleImage()
  ├─ resolveLineContext(lineUserId) → ถ้าไม่พบ → ตอบให้ไป login
  ├─ reply "📷 กำลังอ่าน..." (ใช้ replyToken ก่อนหมดอายุ 1 นาที)
  └─ async: download image → OCR → save LineDraft → push Flex confirm card
  ↓
User กด ✅ "บันทึกเป็นรายจ่าย"
  ↓
webhook: handlePostback() action=confirm
  ├─ reply "⏳ กำลังบันทึก..."
  └─ async confirmDraftAsync():
      ├─ ensureLineDefaults() → get/create Event + Payee
      ├─ download image again → upload to Drive (path: receipts/{org}/{YYYY}/{MM})
      ├─ appendRowByHeaders to Payments sheet (status=pending, มี ReceiptURL + InvoiceFileURL)
      ├─ mark draft confirmed
      ├─ audit log
      └─ push "✅ บันทึกแล้ว" Flex + ลิงก์ /payments
  ↓
User ไปแก้ event/payee/หมวดในเว็บได้ตามปกติ (status=pending → editable)
```

### Defaults สำหรับ LINE Quick Capture
- **Event:** "LINE Quick Capture" (auto-create ครั้งแรก, status=active, budget=0)
- **Payee:** "ไม่ระบุผู้รับเงิน (LINE)" (auto-create, BranchType=HQ)
- **ExpenseType:** "general"
- **Description:** "{vendorName} (จาก LINE)" or "บันทึกจาก LINE"
- **DueDate:** documentDate จาก OCR หรือวันนี้

### Webhook design
- Signature verification ใช้ `verifyWebhookSignature()` (มีอยู่แล้วใน messaging.ts)
- Empty events array → return 200 (LINE Console verify URL)
- ทุก event ผ่าน `Promise.allSettled` — failure แยกกัน
- Always return 200 — LINE retries on non-2xx
