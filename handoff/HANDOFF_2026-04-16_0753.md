## ✅ DONE

### 1. Schema
- `LineDraft.mimeType` (default "image/jpeg") — เก็บ MIME ของ media ที่ส่งมา (ใช้ตอน upload Drive ให้ extension ตรง)
- DB synced ✅ (`npx prisma db push`)

### 2. Handlers
**Renamed:** `handleImage` → `handleMedia` (เก็บ alias backward-compat)

**Now supports:**
- `message.type === "image"` (รูปจากกล้อง/แกลเลอรี LINE)
- `message.type === "file"` ที่นามสกุลเป็น: **pdf, jpg, jpeg, png, webp**

**File flow:**
- Validate extension → reject ทันทีถ้าไม่รองรับ พร้อมแจ้ง user
- Determine MIME จาก filename
- ส่งไป `parseReceipt(buffer, mimeType, "receipt")` — OCR provider เดิมรองรับ PDF อยู่แล้ว (`extractPdfText` ใน `pdf-helper.ts`)
- Save draft พร้อม mimeType
- Confirm step ใช้ extension ตาม MIME (pdf/png/webp/jpg) เวลา upload Drive

### 3. Webhook dispatcher
- Updated `src/app/api/line/webhook/route.ts` — route ทั้ง `image` และ `file` ไป `handleMedia`

### 4. Status feedback message
- Image → "📷 รูปได้รับแล้วค่ะ กำลังอ่าน..."
- PDF → "📄 PDFได้รับแล้วค่ะ กำลังอ่าน..."
